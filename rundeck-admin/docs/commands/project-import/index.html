project-import

Import a project archive.

SYNOPSIS

rerun rundeck-admin:project-import --project <> --url <$RUNDECK_URL> --user <$RUNDECK_USER> --password <$RUNDECK_PASSWORD> [--archive <>] [--retain-uuids <true>] [--import-executions <true>]

OPTIONS

  • –project <>: the project name.
  • –url <$RUNDECK_URL>: rundeck server url.
  • –user <$RUNDECK_USER>: the login user name.
  • –password <$RUNDECK_PASSWORD>: the login password.
  • [–archive <>]: the archive file.
  • [–retain-uuids ]: Create Jobs with their defined UUIDs or a new UUID will be generated for each Job created.
  • [–import-executions ]: Creates new Executions and History reports from the archive.

README

Use project-import to load a project archive to the specified rundeck instance.

Examples

Import a project archive “qa1.archive” to an instance rundeck2:4440:

rerun rundeck-admin:project-import --project qa1 --archive qa1.archive \
    --url http://rundeck2:4440 \
    --user admin --password admin
